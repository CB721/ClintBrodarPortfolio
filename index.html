Back End and Game Development

My biggest challenge in this project was taking on a leadership role. Normally, I am more comfortable in taking instructions instead of making them. I learned how everyone is motivated and should be approached in a different way.

LIRI Node Application

Back End Development

LIRI (Language Interpretation and Recognition Interface) is a command line node app that will search for songs, concerts and movies. Songs are referenced using Spotify's API, concerts are referenced using Bands In Town's API and movies are referenced using OMDB.

"I have so many different apps for finding entertainment. It would be great to have one place I could look for it all."

For Spotify, the user types in node liri.js spotify-this-song 'song name here' and it outputs the following:

  • Artist(s)
  • The song name
  • A preview link of the song
  • The album the song is from

For Bands in Town, the user types in node liri.js concert-this 'artist/band name here' and it outputs the following:

  • Name of the venue
  • Venue location
  • Date of the Event

For OMDB, the user types in node liri.js movie-this 'movie name here' and it outputs the following:

  • Title of the movie
  • Year the movie came out
  • IMDB Rating of the movie
  • Rotten Tomatoes Rating of the movie
  • Country where the movie was produced
  • Language of the movie
  • Plot of the movie
  • Actors in the movie

This was my first project in Node. Working solely in the command line was different at first but, in the end, I found it much more efficient for testing in comparison to using the console log on a browser.

One issue I ran into a couple of times while working on this project was referencing the results from the APIs. In particular, getting the Rotten Tomatoes rating from OMDB was troublesome. I kept getting [object Object] when I referenced it as response.data.Ratings.value. Eventually, I found that I needed to go to the second index in Ratings and get that value.

I challenged myself in this project to implement a log of all the searches in an easily accessible place. The original assignment required that we log the searches to a text file, but I found that difficult to reference since you would have to have access to that text file. To achieve this, I created a Firebase database and had every search logged there.

Web Development

Basketball Trivia Application

Web Development

A multipe-choice guessing game about basketball. Once the user clicks the "Jump Ball" button, they are presented with a list of questions. They then have 24 seconds to answer all of them and hit the "Slam Dunk" button. Either when the user hits the "Slam Dunk" button, or the timer runs out, they are presented with their results.

"I want a game for my kids to play to learn about basketball. These days, they only know about current players and I want them to understand more about the history of the game."

This project taught me about using timing functions in Javascript. I learned how to dynamically create and remove elements from the Html page. Prevously, I would have created a new Html document for each stage of the game.

A major obstacle I faced in this project was the styling. No matter what background or text color I used, the questions and answers blended in to the background. I decided to create a background for the entire div element of the questions and results. This helped separate it from the background.

Train Scheduler

Web Development

Train schedule application that provides up-to-date information about various trains, namely their arrival times and how many minutes remain until they arrive at their station. User can input train information which is then stored in Firebase. This input is then taken from Firebase and displayed for the user.

"I travel often and use public transportation on these trips. It would be great to information for all the stations I visit all located on one website."

Building this application was my first experience using Firebase. Sending the user inputs to Firebase and then displaying them back on the HTML was challenging. I was getting values and seeing them in Firebase, but when I referenced them to put back on the HTML, they would only show up when the user initally clicks the submit button. After troubleshooting for multiple days, I realized I was not referencing the snapshot value when calling the variables.

This project was also my first experience in using moment.js. The most important thing I learned was to create an initial time to reference. Doing this made it possible to create the logic for determining the next arrival time and how many minutes away the next train is.
